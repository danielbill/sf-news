# 想法池 (Idea Backlog)

**用途**: 记录尚未成熟的想法，在创建正式规格前进行快速收集和整理。

---

## 概述

想法池是 spec-kit 工作流中的**第一阶段**，用于在创建完整规格之前快速记录和管理想法。

```
想法池 → 规格 → 计划 → 任务 → 实现
```

---

## 排序规则

想法按 **重要度** 排序，值越大越优先：

| 重要度 | 说明 | 图标 |
|--------|------|------|
| 3 | 🔥 最重要 - 核心功能，必须做 | 🔥 |
| 2 | 📋 中等重要 - 正常功能，按计划做 | 📋 |
| 1 | 💤 不重要 - 有空做，暂不考虑 | 💤 |

---

## 💤 不重要 (重要度1)

---

## 使用方法

```bash
# 添加新想法 (默认重要度2)
/speckit.idea 想法描述

# 指定重要度
/speckit.idea 重要度3 想法描述

# 查看所有想法
/speckit.idea list

# 修改重要度
/speckit.idea set-priority [标题] 重要度3

# 澄清想法 - 回答问题完善内容
/speckit.idea clarify [标题]

# 升级为规格
/speckit.idea promote [标题]
```

## 状态说明

| 状态 | 含义 | 图标 |
|------|------|------|
| 待整理 | 新创建的想法，尚未审核 | 📝 |
| 已确认 | 已审核，等待升级为规格 | 💡 |
| 已完成 | 已升级为规格或已实现 | ✅ |

## 与 spec-kit 工作流的集成

```
/speckit.idea add "描述"
    ↓ 创建 .specify/ideas/想法名称.md
/speckit.idea promote 想法名称
    ↓ 调用 /speckit.specify
/speckit.plan
    ↓ 创建 specs/###-feature/plan.md
/speckit.tasks
    ↓ 创建 specs/###-feature/tasks.md
/speckit.implement
    ↓ 执行实现
```

## 最佳实践

1. **快速记录** - 不要追求完美，先记录下来
2. **定期整理** - 定期 review 想法池，清理或升级想法
3. **设置重要度** - 按重要度排序工作顺序
4. **澄清再升级** - 使用 clarify 完善想法后再升级为规格
