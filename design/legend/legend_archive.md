# Legend 资料归档系统设计方案

## 系统定位

这是一个**知识图谱级别的资料归档系统**，目标是构建「文明前沿雷达系统」的长期记忆层。

**核心目标**：
1. **无限期归档**：从新闻、言论、财报、生态动态等多源采集数据
2. **结构化提取**：将非结构化文本转化为事件、关系、知识图谱
3. **智能查询**：支持 AI 对历史数据进行推理和关联分析

### 系统边界

```
┌─────────────────────────────────────────────────────────────────┐
│                    Legend 归档系统                              │
├─────────────────────────────────────────────────────────────────┤
│  输入层                        输出层                           │
│  ┌─────────────┐              ┌─────────────┐                  │
│  │ 已有新闻源  │──┐           │ 人物档案页  │                  │
│  │ 社交媒体    │──┤           │ 时间线视图  │                  │
│  │ 公司公告    │──┼──→ 归档 ─→│ 关系图谱    │                  │
│  │ 生态动态    │──┘           │ API 查询    │                  │
│  └─────────────┘              └─────────────┘                  │
└─────────────────────────────────────────────────────────────────┘
```

**与现有系统的关系**：
- **独立存储**：`data/legend/` 独立目录，独立 SQLite 库
- **复用采集**：从现有的 Timeline 数据库读取 `legend` 字段打标的新闻
- **扩展采集**：新增言论、财报、生态圈的数据源
